# PR Template

Use this structure for delivery PRs.

## TL;DR

- 1-3 bullets for humans scanning quickly

## Why This Change

- Objective
- why this implementation is needed now

## What Changed

- Group by behavior or subsystem
- explain what changed, where, and why

## Testing

- automated checks run
- manual verification performed
- notable limitations or unrun checks

## QA Notes

- smoke-test prompts or scenarios
- environment or config notes only if actually needed
